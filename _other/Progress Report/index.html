7. Arhitectura aplicatiei

7.1 Descriere

Modelul C4 ia în considerare structurile statice ale unui sistem software în termeni de containere, componente și cod. Iar oamenii folosesc sistemele software pe care le construim.

7.1.1 Person

O persoană reprezintă unul dintre utilizatorii umani ai aplicației web. Aplicatia poate fi utilizata de orice utilizator care are o adresa de mail valida, necesara pentru autentificare si creare de cont.

7.1.2 Software System

Un sistem software este cel mai înalt nivel de abstractizare și descrie ceva care oferă valoare utilizatorilor săi, fie că sunt umani sau nu.

7.1.3 Container

În modelul C4, un container reprezintă o aplicație sau o baza de date. Un container este ceva care trebuie să ruleze pentru ca întregul sistem software să funcționeze.

7.1.4 Component

O componentă este o grupare de funcționalități asociate încapsulate în spatele unei interfețe bine definite.

Nivel 1: Diagrama contextului sistemului

Arată sistemul în domeniul de aplicare și relația acestuia cu utilizatorii și alte sisteme.Diagrama contine in centru sistemul software si este inconjurata de utilizatorii sai si de celelate sisteme cu care interactioneaza: E-mail System, Imgur API, Unsplash Developers API, Graph API.

Utilizatorul poate accesa aplicatia M-Pic, se poate inregistra cu un nou cont sau se poate loga intr-un cont deja existent. Odata inregistrat are acces la functionalitatile aplicatiei web.
